# Deployment Checklist for aioverdose.com

Complete this checklist before deploying to production.

## Pre-Deployment (Local)

### Code Quality
- [ ] Run `npm run build` successfully with no errors
- [ ] Run `npm run dev` and test all pages locally
- [ ] All pages render correctly with no console errors
- [ ] Dark mode toggle works on all pages
- [ ] Responsive design tested on mobile, tablet, desktop
- [ ] All links work (internal and external)
- [ ] Forms can be submitted without JavaScript errors
- [ ] Footer appears on all pages
- [ ] Cookie banner appears and saves preferences to localStorage

### Content & Legal
- [ ] All legal pages are complete:
  - [ ] Privacy Policy (/privacy-policy)
  - [ ] Terms of Service (/terms-of-service)
  - [ ] Cookie Policy (/cookie-policy)
  - [ ] Disclaimer (/disclaimer)
- [ ] Legal pages have contact email (hello@aioverdose.com)
- [ ] Legal pages have last updated dates
- [ ] Contact page form is functional
- [ ] Newsletter form saves to provider (TODO: integrate with Substack/ConvertKit)

### Features Tested
- [ ] Audit tool loads and accepts URL input

- [ ] Results display with proper styling
- [ ] Score circle animation works
- [ ] Category cards are expandable

### Trust & Security
- [ ] TrustBar displays on homepage

- [ ] Contact form collects name, email, subject, message
- [ ] Cookie banner has three options: Accept All, Reject All, Customize

### SEO & Performance
- [ ] Meta descriptions are present and under 160 characters
- [ ] Page titles are descriptive (30-60 characters)
- [ ] Open Graph tags are set up

- [ ] Robots.txt configured at `/robots.txt`
- [ ] Organization schema markup present in layout

### Analytics & Tracking
- [ ] Vercel Analytics configured
- [ ] (Optional) Plausible Analytics account created
- [ ] Cookie consent integrates with analytics

## Rollback Plan

If issues occur in production:

1. **Critical Issue Detected**
   - [ ] Check Vercel deployment logs for errors
   - [ ] Roll back to previous deployment: Vercel → Deployments → Previous version → Promote

2. **Fix Locally**
   - [ ] Identify root cause
   - [ ] Create hotfix branch
   - [ ] Test locally with `npm run build && npm run start`
   - [ ] Commit and push

3. **Re-Deploy**
   - [ ] Vercel will automatically redeploy on push to main
   - [ ] Verify production site is healthy
   - [ ] Test critical paths

## Final Verification Script

Before marking deployment as complete, run this script:

```bash
#!/bin/bash
echo "🔍 Running final verification..."
echo ""

# Build
echo "📦 Building..."
npm run build || exit 1

# Check required files
echo "📄 Checking required files..."
test -f ".next/server/sitemap.js" && echo "✓ Sitemap found" || echo "✗ Sitemap missing"
test -f ".next/server/robots.js" && echo "✓ Robots.txt found" || echo "✗ Robots.txt missing"

# Check key pages
echo "📄 Checking key pages..."
test -d "app/tools/audit" && echo "✓ Audit page found" || echo "✗ Audit page missing"
test -d "app/contact" && echo "✓ Contact page found" || echo "✗ Contact page missing"
test -d "app/newsletter" && echo "✓ Newsletter page found" || echo "✗ Newsletter page missing"
test -d "app/resources" && echo "✓ Resources page found" || echo "✗ Resources page missing"

# Check legal pages
echo "📋 Checking legal pages..."
test -d "app/privacy-policy" && echo "✓ Privacy policy found" || echo "✗ Privacy policy missing"
test -d "app/terms-of-service" && echo "✓ Terms of service found" || echo "✗ Terms of service missing"
test -d "app/cookie-policy" && echo "✓ Cookie policy found" || echo "✗ Cookie policy missing"
test -d "app/disclaimer" && echo "✓ Disclaimer found" || echo "✗ Disclaimer missing"

# Check components

test -f "components/Footer.tsx" && echo "✓ Footer found" || echo "✗ Footer missing"
test -f "components/CookieBanner.tsx" && echo "✓ CookieBanner found" || echo "✗ CookieBanner missing"
```
